## 已排除内容

- `node_modules`：依赖缓存，体积大，可通过 `npm install` 重新生成。
- `target`：Maven 构建产物，可通过 Maven 重新生成。
- `dist`、`unpackage`：前端和 uni-app 构建产物，不作为源码维护。
- `.idea`、`.hbuilderx`：本机 IDE 配置，不适合作为团队默认配置。
- `logs`、`*.log`：运行日志，不进入新主项目。
- `ISLab\ISLab`：外层 ISLab 是当前实际使用版本，内层缺少新联调页面和脚本，判断为旧副本。
- `mobile-app\uni-shop-2`：嵌套的旧模板/副本，当前主 app 未引用。
- `ISLab (平台项目源码)new`：额外源码包副本，不作为当前维护入口。

## 新目录组织原因

- `apps` 放前端和 app，便于后续继续增加客户端。
- `server` 放后端主工程，保持若依模块结构不变。
- `database` 单独放 SQL，便于部署交付。
- `scripts` 统一放启动、停止、冒烟测试脚本，避免散落在子项目中。
- `docs` 放迁移、环境、运行说明，作为后续团队协作入口。
